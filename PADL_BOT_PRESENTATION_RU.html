PADL BOT

Презентация по работе программы

Как Telegram-бот следит за свободными слотами PADL, фильтрует подходящие варианты и сообщает пользователю, когда можно записаться.

PowerShell
.\.venv\Scripts\Activate.ps1
python -m padlbot

Telegram
/search
Ключевая идея

Бот не бронирует слот автоматически. Он постоянно мониторит доступность и отправляет уведомление, а пользователь записывается вручную на сайте PADL.

Задача

Проблема и сценарий пользователя

Свободные слоты появляются нерегулярно. Пользователю неудобно постоянно обновлять сайт и проверять все площадки вручную.

мониторинг вместо ручной проверки
1

Нужно место

Пользователь хочет найти слот для игры в падел.

2

Слоты меняются

Доступность на сайте может появиться в любой момент.

3

Бот проверяет

Программа регулярно обращается к API PADL.

4

Пользователь реагирует

Когда бот присылает слот, запись выполняется вручную на сайте.

Главная польза: бот экономит внимание. Он не заменяет сайт PADL, а помогает быстрее заметить новые свободные варианты.

Запуск

Как программа стартует

Для работы нужны Python, зависимости, Telegram-токен в `.env` и команда запуска из папки проекта.

python -m padlbot
1. Окружение
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

2. Запуск
python -m padlbot
Что происходит внутри
  • Читаются настройки из `.env`.
  • Открывается SQLite-база `data/padlbot.db`.
  • Создается клиент Telegram и клиент API PADL.
  • Проверяется, что не запущена вторая копия бота.
  • Запускается цикл получения сообщений из Telegram.

Окно PowerShell должно оставаться открытым. Пока команда работает, бот принимает команды и может вести мониторинг.

Telegram

Основные команды

Пользователь управляет ботом через короткие команды. Основная команда для работы - `/search`.

управление без интерфейса сайта
/start

Справка

Показывает краткое описание и ссылку на сайт PADL.

/search

Мониторинг

Запускает постоянную проверку свободных слотов.

/now

Проверка сейчас

Показывает актуальные слоты без остановки мониторинга.

/status

Статус

Сообщает, активен ли мониторинг и что было найдено.

/stop

Останавливает активную фоновую задачу поиска.

Если бот не отвечает, первое действие пользователя - открыть бота в Telegram и нажать Start или отправить `/start`.

Мониторинг

Цикл поиска свободных слотов

После `/search` бот создает фоновую задачу. Она повторяется, пока пользователь не отправит `/stop`.

примерно каждые 15 секунд
1

Прочитать настройки

Берутся сохраненные предпочтения пользователя или значения по умолчанию.

2

Запросить API PADL

Бот проверяет площадки и доступные события через `date-options`.

3

Отфильтровать слоты

Оставляются варианты с нужным числом мест и подходящей длительностью.

4

Найти новые

Слоты, которые уже отправлялись во время текущего запуска, не дублируются.

5

Отправить сообщение

Если есть новые варианты, бот присылает их в Telegram со ссылкой на сайт.

6

Подождать и повторить

После паузы цикл начинается снова. Временные ошибки не останавливают бота.

Состояние

Что хранится в SQLite

Локальная база помогает боту помнить настройки и восстанавливать активный мониторинг после перезапуска.

data/padlbot.db
profiles

Старый профиль пользователя: имя, телефон и email. Для текущего `/search` обычно не нужен.

preferences

Параметры поиска: площадки, длительности, количество мест, интервал проверки.

search_state

Активен ли мониторинг и какой последний статус был записан.

last_bookings

Legacy-данные о последней подтвержденной брони, если использовался старый сценарий.

Во время одного активного запуска список уже отправленных слотов хранится в памяти, чтобы бот не присылал одно и то же повторно.

Это важная часть работы программы: текущий режим создан для мониторинга и уведомлений, а не для автоматической записи.

Диагностика

Частые ошибки и как их читать

Большинство проблем связано с настройками запуска, Telegram-токеном или тем, что бот уже запущен где-то еще.

быстрая проверка
`TELEGRAM_BOT_TOKEN is required`

В `.env` нет токена или команда запущена не из папки проекта.

`chat not found`

Пользователь еще не нажал Start в Telegram-боте.

`Another bot instance is already running`

На этом компьютере уже работает другая копия процесса.

Бот молчит

Проверить интернет, токен, открытое окно PowerShell и отсутствие второй копии.

Нужно проверить без Telegram

Запустить `python scripts\dry_run.py` и посмотреть, найден ли подходящий слот.

Временная ошибка API

Бот записывает статус ошибки и пробует снова после следующей паузы.
