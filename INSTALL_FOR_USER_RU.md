# Инструкция Для Установки И Настройки PADL BOT

Эта инструкция нужна, если бот переносится на другой компьютер или другому человеку
нужно запустить собственную копию бота.

Бот мониторит свободные слоты PADL на `outdoor.sport.mos.ru` и отправляет уведомления
в Telegram. Он не бронирует слот автоматически, не удерживает его, не обходит СМС и
не подтверждает запись. После уведомления записываться нужно вручную на сайте PADL.

## 1. Что Нужно Заранее

Установите:

- Python 3.10 или новее.
- Git, если проект будет скачиваться из репозитория. Если проект передают архивом,
  Git не обязателен.
- Telegram-бот, созданный через `@BotFather`.

У `@BotFather` нужно получить токен вида:

```text
123456789:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Если используется уже существующий бот, возьмите его текущий токен у владельца.
Не запускайте две копии бота с одним и тем же Telegram-токеном одновременно на разных
компьютерах: Telegram polling может начать конфликтовать.

## 2. Получить Проект

Вариант с архивом:

1. Скопируйте папку проекта `PADL BOT` на компьютер пользователя.
2. Откройте PowerShell.
3. Перейдите в папку проекта:

```powershell
cd "C:\path\to\PADL BOT"
```

Вариант с Git:

```powershell
git clone ССЫЛКА_НА_РЕПОЗИТОРИЙ "PADL BOT"
cd "PADL BOT"
```

## 3. Установить Зависимости

Создайте виртуальное окружение:

```powershell
python -m venv .venv
```

Активируйте его:

```powershell
.\.venv\Scripts\Activate.ps1
```

Если PowerShell запрещает запуск скриптов, выполните один раз:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

После этого снова активируйте окружение:

```powershell
.\.venv\Scripts\Activate.ps1
```

Установите зависимости:

```powershell
pip install -r requirements.txt
```

## 4. Настроить `.env`

Скопируйте пример настроек:

```powershell
Copy-Item .env.example .env
```

Откройте файл `.env` в любом редакторе и заполните:

```env
TELEGRAM_BOT_TOKEN=токен_от_BotFather
ADMIN_CHAT_ID=
PADL_DB_PATH=data/padlbot.db
PADL_SITE_BASE_URL=https://api.outdoor.sport.mos.ru
AUTO_START_SEARCH=0
SMS_WEBHOOK_HOST=0.0.0.0
SMS_WEBHOOK_PORT=8080
REQUEST_TIMEOUT_SECONDS=15
```

Обязательно:

- `TELEGRAM_BOT_TOKEN` - токен Telegram-бота.

Опционально:

- `ADMIN_CHAT_ID` - Telegram chat id пользователя. Обычно можно оставить пустым.
- `AUTO_START_SEARCH=1` - если нужно, чтобы бот сам запускал мониторинг при старте.
  Для этого желательно указать `ADMIN_CHAT_ID`.
- `REQUEST_TIMEOUT_SECONDS=15` - таймаут запросов к API PADL.

Для обычного режима мониторинга `SMS_FORWARD_SECRET`, Android SMS Forwarder и профиль
пользователя не нужны.

## 5. Запустить Бота

Из папки проекта, с активированным `.venv`, выполните:

```powershell
python -m padlbot
```

Окно PowerShell должно оставаться открытым. Пока команда работает, бот принимает
команды из Telegram.

Чтобы остановить бота, нажмите:

```text
Ctrl+C
```

## 6. Проверить Работу В Telegram

1. Откройте Telegram.
2. Найдите своего бота по username.
3. Нажмите `Start` или отправьте:

```text
/start
```

4. Бот должен показать одну основную команду:

```text
Запустить постоянный мониторинг: /search
Бот ищет без ограничения по времени.
Записывайтесь вручную на сайте PADL:
https://outdoor.sport.mos.ru/#venues-events
```

5. Запустите мониторинг:

```text
/search
```

По умолчанию бот ищет:

- площадки PADL: Баррикадная, Третьяковская, Римская;
- формат: `Свободная игра`;
- время: без ограничения;
- количество мест: 2;
- длительность по приоритету: 120, 90, 60 минут;
- интервал проверки: примерно 15 секунд.

## 7. Основные Команды

```text
/start
```

Показать краткую справку.

```text
/search
```

Запустить постоянный мониторинг без ограничения по времени. Когда бот пришлёт свободный слот,
записывайтесь вручную на сайте PADL:

```text
https://outdoor.sport.mos.ru/#venues-events
```

```text
/now
```

Показать свободные слоты прямо сейчас, не останавливая мониторинг.

```text
/status
```

Показать текущий статус.

```text
/stop
```

Остановить мониторинг.

## 8. Проверка Из Терминала

Можно проверить поиск свободных слотов без Telegram:

```powershell
python scripts\dry_run.py
```

Если слот найден, команда выведет его в терминал. Если подходящих слотов нет, будет:

```text
No matching slot found.
```

## 9. Частые Проблемы

`TELEGRAM_BOT_TOKEN is required`

В `.env` не указан `TELEGRAM_BOT_TOKEN` или команда запущена не из папки проекта.

`Telegram API error: chat not found`

Пользователь ещё не нажал `Start` в Telegram-боте. Нужно открыть бота и отправить
`/start`.

Бот не отвечает

Проверьте, что:

- PowerShell с командой `python -m padlbot` открыт и команда не завершилась;
- указан правильный Telegram-токен;
- нет второй копии бота с тем же токеном;
- компьютер имеет доступ к интернету.

`Only one PADL bot instance can run...`

На этом компьютере уже запущена другая копия бота. Закройте старый процесс или
перезагрузите компьютер.

## 10. Перенос На VPS

На VPS действия такие же:

1. Установить Python 3.10+.
2. Скопировать проект.
3. Создать `.venv`.
4. Установить `pip install -r requirements.txt`.
5. Создать и заполнить `.env`.
6. Запустить:

```bash
python -m padlbot
```

Для постоянной работы на VPS лучше запускать бота через `systemd`, `screen`, `tmux`
или другой менеджер процессов.
