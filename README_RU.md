# PADL BOT

Telegram-бот для постоянного мониторинга бесплатных слотов падела на `outdoor.sport.mos.ru`.
Для запуска используется `aiohttp`: через него бот работает и с Telegram, и с API сайта.

## Что Делает Бот

- Проверяет все три площадки для формата `Свободная игра`.
- По умолчанию ищет любой доступный день в горизонте сайта, время `17:00-22:00`, 2 места, длительность по приоритету `120 -> 90 -> 60` минут.
- Если появляются новые подходящие слоты, присылает их в Telegram.
- Продолжает мониторинг после уведомления и не повторяет один и тот же слот во время активного запуска.
- Не удерживает слот, не запрашивает СМС и не подтверждает бронь из `/search`; записываться нужно вручную на сайте PADL.

## Локальная Установка

На этой машине `.venv` уже создан. Для запуска здесь достаточно:

```powershell
.\.venv\Scripts\Activate.ps1
python -m padlbot
```

Если переносите проект на другой компьютер, используйте полную установку:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
```

Откройте `.env` и заполните:

- `TELEGRAM_BOT_TOKEN` - токен бота от BotFather.
- `ADMIN_CHAT_ID` - ваш Telegram chat id. Полезно, если SMS-forwarder не отправляет `chat_id`.
- `SMS_FORWARD_SECRET` и `PADL_DRY_RUN` не нужны для `/search`: мониторинг всегда только уведомляет.

Запуск:

```powershell
python -m padlbot
```

## Команды В Telegram

- `/start` - показать подсказку по настройке.
- `/profile ИМЯ ФАМИЛИЯ ТЕЛЕФОН EMAIL` - сохранить старый профиль, если он нужен для legacy SMS-команд.
- `/search 17:00-22:00` - начать постоянный мониторинг на любой доступный день в горизонте сайта.
- `/search 2026-06-12 17:00-22:00` - мониторить слот на конкретную дату.
- `/now` - показать слоты, доступные прямо сейчас, не останавливая мониторинг.
- `/status` - показать текущий статус и последнюю бронь.
- `/stop` - остановить активный мониторинг.
- `/code 1234` - legacy-команда для ручной отправки СМС-кода, если слот уже был удержан старым сценарием.

## Legacy Android SMS Forwarder

Новому режиму `/search` пересылка СМС не нужна. Этот endpoint оставлен только для
старых сценариев, где слот уже был удержан.

Для локальной проверки телефон и компьютер должны быть в одной Wi-Fi сети. Настройте Android-forwarder на POST-запрос:

```text
http://LOCAL_IP_ВАШЕГО_ПК:8081/sms
```

Тело JSON:

```json
{
  "secret": "значение-из-SMS_FORWARD_SECRET",
  "text": "текст СМС с кодом 4821",
  "chat_id": 123456789
}
```

Если в `.env` указан `ADMIN_CHAT_ID`, поле `chat_id` можно не отправлять.

## Проверка Доступности Из Терминала

Команда `/search` всегда только уведомляет о новых свободных слотах и продолжает мониторинг до `/stop`.

Если `AUTO_START_SEARCH=1`, бот сам начнёт мониторинг при запуске.

Проверка свободных слотов из терминала:

```powershell
python scripts\dry_run.py
```

## Перенос На VPS

На VPS установите те же зависимости, перенесите `.env` и запустите `python -m padlbot`.

Бот не обходит капчи, СМС, лимиты или закрытые механизмы сайта. Он только читает доступность через API и сообщает о свободных слотах, а запись выполняется вручную на сайте PADL.
